Estoy iniciando un sistema de facturación local para Argentina utilizando C++20 en un entorno Linux (Mint) con OpenCode. Quiero diseñar la lógica del negocio de forma modular y limpia, aplicando buenas prácticas modernas, antes de meter la base de datos o la interfaz visual.

    Requerimientos del Negocio (Argentina):

        Modelos de Datos: Necesito las estructuras/clases básicas para Cliente (Nombre, CUIT/CUIL, Condición de IVA), Producto (Código, Descripción, Precio Neto, Alícuota de IVA: 21%, 10.5%, 0%), y Factura (Número, Tipo A/B/C, Fecha, Cliente, Items, Subtotal Neto, Total IVA, Total Final).

        Reglas de Negocio:

            Si el cliente es "Responsable Inscripto", la factura debe ser Tipo A (discriminando el IVA).

            Si el cliente es "Consumidor Final" o "Monotributista", la factura debe ser Tipo B o C (el IVA se incluye en el precio neto/precio final expuesto).

        Validación: Una función auxiliar para validar que un CUIT (string de 11 dígitos) cumpla con el algoritmo del dígito verificador de AFIP.

    Requerimientos Técnicos (C++20):

        Estructura modular: Divide el código claramente indicando qué va en los archivos de cabecera (.hpp) y qué en los archivos de implementación (.cpp).

        Usa características modernas de C++20 (como std::optional, std::string_view, inicialización directa de miembros, o contenedores eficientes como std::vector).

        No agregues ninguna librería gráfica (GUI) ni código de base de datos por ahora. Todo debe resolverse con lógica pura y salidas limpias por consola (std::cout) para testing.

        Incluye una función main() de prueba simple que cree un cliente, dos productos, calcule una factura Tipo A y otra Tipo B, y las imprima formateadas en la terminal.

    Genera el código de forma estructurada, archivo por archivo, explicando brevemente la responsabilidad de cada uno.
